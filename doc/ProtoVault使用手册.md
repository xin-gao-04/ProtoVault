# ProtoVault 使用手册与设计思路

一句话模型：ProtoVault 把 C++ Header 里的数据协议，提升为可扫描、可编辑、可检查、可生成文档、可分析网络数据流的工程资产。

界面内的“?”入口现已升级为“AI 使用助手”。它读取模块化功能知识库，并通过本地 Ollama 模型回答操作问题；Ollama 不可用时会降级为离线知识库摘要。

## 1. 推荐工作流

1. 打开本地工作区或加载示例项目。
2. 等待 Header 扫描完成，检查问题面板和右侧属性栏。
3. 在左侧协议树中定位 Header、Struct、Enum、字段或枚举项。
4. 在中间表格直接编辑字段、枚举值、注释和初始化值。
5. 使用协议关系图谱查看 Header / 类型之间的包含与引用关系。
6. 使用网络地图维护实体节点、通信链路和协议绑定。
7. 先在“数据流视角”中定义观察范围，再进入“数据流画布”查看生产节点、链路载荷和消费节点。
8. 通过 Lint、文档、Git 基线 Tag 和版本 Diff 检查协议演进风险。

## 2. 图谱与数据流的职责边界

ProtoVault 不再把所有关系都塞进一个图里。

| 入口 | 主要回答的问题 | 不负责的问题 |
|---|---|---|
| 协议关系图谱 | Header、Struct、Enum 如何包含和引用 | 系统数据链路如何传输 |
| 网络地图 / 节点 | 系统里有哪些实体节点 | 协议字段怎么编辑 |
| 网络地图 / 链路 | 节点之间如何通信 | Header 类型依赖 |
| 网络地图 / 协议绑定 | 哪个协议在某条链路上传输 | 业务视角筛选 |
| 数据流视角 | 当前想观察哪一组节点、链路和协议 | 直接表达完整拓扑事实 |
| 数据流画布 | 数据从生产节点经链路载荷流向消费节点 | 替代结构化编辑表格 |

设计原则：节点和链路是事实层，协议绑定是传输事实，数据流视角是观察层。大型仿真项目往往有很多业务数据流，但它们共享同一张数据链路网络，因此应先维护网络事实，再派生业务视角。

## 3. 链路字段解释

### 延迟预算

延迟预算是用户或团队录入的设计上限，不是当前自动测量值。

例如某条 DDS 链路希望 20 ms 内完成发送、调度、接收和解析，就可以把延迟预算写成 `20`。当前版本主要用它服务设计审查和文档化；后续接入运行期观测后，可以把实测延迟和预算做对比。

### 带宽上限

带宽上限表示链路可接受的吞吐边界。数据流画布会用估算带宽和上限比较，帮助标记潜在瓶颈。

### 峰值系数

峰值系数属于协议绑定，用于把平均吞吐放大成突发吞吐估计。

- `1.0`：不放大。
- `2.0`：按平均吞吐两倍估算。
- 大于 `2.0`：视为明显突发负载，画布会倾向标记为关注项。

## 4. 常用快捷键

| 快捷键 | 行为 |
|---|---|
| `Ctrl + S` | 保存当前 tab 的源码、结构化编辑、注释或相关脏数据 |
| `F2` | 编辑当前选中的 Header、类型、字段或枚举项 |
| `Alt + ←` | 返回上一步界面 |
| `Alt + →` | 前进到下一步界面 |

全局后退 / 前进是界面导航历史，不等同于撤销编辑。它只恢复用户所在的位置和视图，不主动修改文件。

## 5. Git 基线与版本 Diff

ProtoVault 使用 Git 分支和 Tag 表达协议演进，而不是维护一套孤立的快照体系。

- 分支：适合表示实验线、任务线或发布维护线。
- Tag：适合表示一次可追溯的协议基线，例如 `protovault/baseline/20260705T120000Z`。
- Git 状态：显示在左侧底部工作区栏，包含当前分支、最近 Tag 和 dirty 改动数量。
- 源代码管理入口：左侧工作栏的“源代码管理 / Git”按钮。
- 提交入口：进入源代码管理视图后，暂存文件或全部暂存，填写 Git 提交信息，然后点击“提交暂存更改”。
- 分支入口：源代码管理视图中可以切换已有本地分支，也可以新建并切换分支。
- 基线 Tag：创建前要求当前工作区没有未提交改动，避免 Tag 指向的提交和协议文件实际状态不一致。
- 基线 Tag 入口：顶部工作栏的“基线 Tag”按钮。
- 版本 Diff 入口：顶部工作栏的“版本 Diff”按钮。
- 版本 Diff：默认比较最近基线 Tag 与当前工作树，报告字段、枚举、布局、网络节点、链路、协议绑定和 FlowView 变化。

当前版本暂不提供 push、pull、远端认证和放弃更改操作。完成一组协议变更并通过 Lint / 文档检查后，建议在源代码管理视图中提交，再创建协议基线 Tag。这样后续 Diff、回溯和发布说明都有稳定锚点。

## 6. Tab 习惯

- 单击左侧树节点进入预览 tab。
- 双击树节点会固定为正式 tab。
- 源码、结构化字段、枚举项或注释存在未保存改动时，tab 会显示脏状态。
- 关闭存在脏状态的 tab 时会提示，避免静默丢失修改。

## 7. 当前排除或延后的内容

以下能力仍不是当前 MVP 的重点：

- 运行期实测延迟采集。
- 数据流画布中的拖拽式编辑。
- 自动瓶颈结论生成。
- 多人协作审批。
- 完整 C++ 语法、复杂宏和模板编辑。

一句话总结：ProtoVault 的界面应围绕“协议结构编辑”和“网络事实建模”分工，关系图谱看类型依赖，数据流画布看传输路径，避免多个图重复表达同一个问题。
