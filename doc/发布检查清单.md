# ProtoVault 发布检查清单

本文档记录当前 P11/P18 发布门的最小可执行标准。现阶段目标是保证开发态、测试态和 Windows `v0.2.0` 独立分发包都可以重复验证。

## 发布前必须通过

```powershell
pnpm release:check
```

该脚本会顺序执行：

1. TypeScript 类型检查。
2. 单元测试。
3. Electron/Vite 构建。
4. Playwright Electron E2E。
5. C++ core CMake configure/build/CTest。

## 构建 Windows 初始分发包

```powershell
pnpm release:installer
```

当前会生成：

- `apps/desktop/release/ProtoVault-0.2.0-Setup-x64.exe`
- `apps/desktop/release/ProtoVault-0.2.0-Portable-x64.exe`

分发产物不提交到 Git，统一由 Release 流程上传或本地分发。

`release:installer` 使用当前项目已经安装的 Electron 运行时，避免打包阶段再次在线下载 Electron。Git/Clang 仍需在构建机存在一次，用于准备随包工具链；这是构建要求，不是目标机器运行要求。

## 手工验收

- 启动桌面端后能加载 `examples` 示例工作区。
- 能打开真实目录工作区，并递归展示 Header、空目录、Struct、Enum、字段和枚举项。
- 字段/枚举表格支持内联编辑、右键删除、`Ctrl+S` 保存。
- Header Source tab 支持源码编辑、保存后重扫、解析错误修复。
- 外部修改 Header 后出现冲突提示；存在本地草稿时可选择保留草稿导入或舍弃草稿导入，并可查看版本影响。
- Lint、文档、Git 基线 Tag、版本 Diff 可以生成报告。
- 全局主题切换会影响表格、属性栏和关系图谱。
- 工作区底栏显示 SQLite 索引命中与新解析数量；重启后 warm scan 能复用缓存。
- 在只含 `C:\Windows\System32;C:\Windows` 的 PATH 下启动打包版，加载全新工作区并验证随包 Clang/Git。

## 目标机器依赖

- Setup 与 Portable 均内置 Electron、Git for Windows、Clang 和 MVP 标准头兼容层。
- 目标 Windows 10/11 x64 机器不需要安装 Node.js、pnpm、Git、LLVM/Clang 或 Visual Studio。
- “AI 使用助手”的在线本地模型回答是可选能力；只有该能力需要用户另行启动 Ollama 和本地模型。Ollama 不可用时仍提供离线知识库回答。

## 当前尚未进入正式发布包的事项

- 还没有配置正式应用图标、代码签名证书和自动更新通道。
- C++ protocol-core 仍是服务骨架，解析逻辑目前在 Electron 主进程中调用 Clang AST。
- 当前已有 SQLite Header IR cache、稳定身份和持久最后有效 IR；完整 Canonical IR/C++ Parser Service 收敛仍未完成。
- 冲突处理已有草稿保留/舍弃与 Diff 入口，但尚未提供逐行 Disk/Base/Draft 三方可视合并。
- 未签名安装包在部分 Windows SmartScreen 策略下可能显示发布者未知；这不影响离线运行，但正式公开发布前仍需代码签名证书。
