# ProtoVault 发布检查清单

本文档记录当前 P11 发布门的最小可执行标准。现阶段目标是保证开发态产物可重复验证；正式安装包仍作为后续发布工程任务推进。

## 发布前必须通过

```powershell
pnpm release:check
```

该脚本会顺序执行：

1. TypeScript 类型检查。
2. 单元测试。
3. Electron/Vite 构建。
4. Playwright Electron E2E。
5. C++ core CMake configure/build/CTest。

## 手工验收

- 启动桌面端后能加载 `examples` 示例工作区。
- 能打开真实目录工作区，并递归展示 Header、空目录、Struct、Enum、字段和枚举项。
- 字段/枚举表格支持内联编辑、右键删除、`Ctrl+S` 保存。
- Header Source tab 支持源码编辑、保存后重扫、解析错误修复。
- 外部修改 Header 后出现冲突提示，可重新扫描或查看 Diff。
- Lint、文档、Git 基线 Tag、版本 Diff 可以生成报告。
- 全局主题切换会影响表格、属性栏和关系图谱。

## 当前尚未进入正式发布包的事项

- 还没有接入 electron-builder / Electron Forge 生成 Windows 安装包。
- C++ protocol-core 仍是服务骨架，解析逻辑目前在 Electron 主进程中调用 Clang AST。
- 大工作区还没有 SQLite 增量索引和持久 AST 缓存。
- 冲突处理面板当前提供重新扫描与 Diff，不提供自动覆盖生成。
