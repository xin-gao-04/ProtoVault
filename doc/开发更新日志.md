# ProtoVault 开发更新日志

本文档记录 Codex Agent 在本地推进 ProtoVault MVP 的实际交付、验证结果和已知限制。它用于配合 Git 提交历史，让后续开发能从“做到了什么、为什么这么做、下一步是什么”继续推进。

## 2026-07-01

### 工程与运行环境

- 建立 pnpm workspace，包含桌面端、契约包和 C++ 服务骨架。
- 桌面端使用 Electron、React、TypeScript、Vite。
- C++ 服务使用 CMake，当前以健康检查和测试骨架作为 P0/P1 基础。
- Windows 本机验证通过：VS Build Tools、LLVM/Clang、Electron、Vitest、Playwright。

### 工作区扫描与 Header 解析

- 支持打开任意文件夹作为工作区，不再要求必须直接打开 `headers/` 根目录。
- 递归发现 `.h/.hh/.hpp/.hxx` 文件，并保留空目录到前端树模型。
- 扫描时跳过 `.git`、`.protocol`、`node_modules`、`out`、`dist`、`coverage`、`build*` 等噪声目录。
- 使用 Clang JSON AST 提取 namespace、struct、enum、字段、定长数组和源码位置。
- 修复 include 声明归属错误：`includedFrom` 仅代表包含来源，不能作为声明所在文件。因此 `geometry.hpp` 中的 `Vec3`、`CoordinateFrame` 不再错误挂到 `track.hpp` 下。

### 目录记录

- 每次扫描后写入 `.protocol/workspace.json`。
- 记录内容包括工作区名称、根路径、扫描时间、目录、Header、类型、诊断和计数。
- 写入采用临时文件 + 原子替换，避免半写入状态污染工作区。

### UI 与交互

- 左侧导航已调整为 Obsidian 风格的合并树：目录 → Header → struct/enum → 字段/枚举项。
- 支持目录、Header、类型节点独立展开和收起。
- 支持三栏拖拽调整：导航栏、主工作区、属性栏。
- 工作区打开与示例扫描放到底部 dock，协议操作放在顶部小图标区域。
- 主工作区支持 Header 源码预览、struct 字段表、enum 值表、属性面板和诊断面板。

### Obsidian/Tokyo Night 主题兼容

- 添加 Obsidian CSS variables 兼容层。
- 添加 `user-obsidian-theme.css` 作为本地主题接入点。
- 当前接入本机主题：

```text
D:\my-documents\.obsidian\themes\Tokyo Night\theme.css
```

- ProtoVault 会优先使用 Tokyo Night 的 `--bg`、`--fg`、`--cyan`、`--blue`、`--comment` 等变量，并映射到通用 Obsidian token，例如 `--background-primary`、`--text-normal`、`--interactive-accent`。

### 基础结构化编辑

- 顶部小图标已接入第一版真实写入动作：
  - 新建 Header。
  - 在当前 Header 创建 `struct`。
  - 给选中的 `struct` 追加字段。
- 写入后自动重新扫描工作区并刷新树、源码预览、类型详情和 `.protocol/workspace.json`。
- 写入路径限制在当前 workspace 内。
- Header/record 写入使用原子替换。

### 当前已知限制

- 结构化编辑 UI 暂时使用 `prompt`，后续应迁移到右侧属性面板表单。
- `addField` 当前支持简单 `struct Name { ... };` 形式，不处理复杂宏、继承、模板或带特殊格式的声明块。
- Header 生成仍是最小可用版本，尚未实现完整确定性代码生成器、格式化策略和冲突面板。
- Obsidian 主题兼容以 CSS variables 为主，不承诺完整复用 Obsidian DOM 选择器效果。

### 下一步建议

1. 将新建 Header / struct / field 的 `prompt` 升级为正式表单。
2. 给写入动作增加预览和撤销入口。
3. 建立 IR → Header 的确定性生成器，并让结构化编辑走受控生成路径。
4. 添加文件哈希基线和外部修改冲突检测，进入 P7 双向同步风险门。

## 2026-07-01 补充推进

### 界面布局

- 左侧协议树默认宽度从 430px 收缩到 340px。
- 左侧树拖拽范围调整为 280px–560px，让主工作区获得更大的默认阅读空间，同时保留用户手动放宽树栏的能力。

### 示例工作区

- 扩充 `examples/radar-workspace`：
  - `common/geometry.hpp` 增加 `Quaternion`、`Pose3D` 和更多坐标系枚举项。
  - 新增 `common/time.hpp`，包含 `Timestamp`、`SequenceId`、`QualityLevel`。
  - `radar/track.hpp` 增加 `TrackState`、时间戳、状态字段和 `TrackCluster`。
  - 新增 `radar/detection.hpp`，包含 `DetectionSource`、`RadarDetection`、`DetectionFrame`。
- 目的：让左侧树、字段表、枚举表和 include 关系更接近真实协议项目。

### 工作区记忆

- Electron 主进程在 `userData/preferences.json` 记录 `lastWorkspacePath`。
- 用户打开本地目录或加载示例项目后，应用会记住该工作区。
- 下次启动自动尝试扫描并恢复上次工作区。
- E2E 测试通过 `PROTOVAULT_DISABLE_RESTORE=1` 禁用自动恢复，避免测试受本机偏好影响。
